## New items from v0.4 ship verification (2026-04-28 / 2026-04-29)

### Paywall PDF workflow polish

- [ ] **Unpaywall fallback in `pdf_extract.fetch_pdf`** — for paywalled venues (IEEE TII, Elsevier ESWA, etc.) where `paper.pdf_url` is empty or returns a DOI-gateway HTML page, query Unpaywall by DOI to find legitimate OA copies (author self-archive). ~30 min work; expected to recover 30–50% of paywall papers automatically without manual import. Falls through to the existing `import_pdf` path for the remaining %.

- [ ] **Roman-numeral heading regex** — `pdf_extract._RE_INTRO/_METHOD/_LIMITS` currently match Arabic-numbered headings (`1. Introduction`). IEEE TII papers use Roman (`I. INTRODUCTION`), causing fallback (head 6K + tail 2K) instead of clean section slicing. Add `[IVX]+\.?` to the heading prefix group. ~5 min; low priority since fallback already produces useful context (~8K chars/paper observed in manufacturing topic run).

### Topic-specific venue weight presets

- [ ] **Per-topic `_VENUE_WEIGHT` overrides** — currently global. Different topics deserve different priors — manufacturing/applied-AI work benefits from TII/ESWA=5, while LLM/diffusion topics want top-tier conferences=5 and TII/ESWA=2. Surface as `--venue-preset {default|applied|nlp|cv}` or read from a topic-keyed config file. Avoids per-topic global weight edits like the manual 3↔5 toggle done during v0.4 ship verification.

### Mixed-model bot routing (carry-over from v0.4 I-3 stretch)

- [ ] **Route Trend + Gap to gemini-flash, Skeptic + Proposer to gemini-pro/sonnet** — cheap pattern-matching on the front half, sharper reasoning on the back half. Empirically saves 50–70% even without full migration. Single-model default is fine for now but worth revisiting if cost or output-quality balance shifts.
