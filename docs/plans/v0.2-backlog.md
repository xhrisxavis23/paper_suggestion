- [ ] **I-3**: Robust import path for `match_substring.py` and `build_report.py`. They currently rely on `python -m` invocation. Either add the `_REPO_ROOT` sys.path bootstrap (as in `load_metadb.py:9-12`) and switch to absolute imports, or document that direct script invocation is unsupported.

- [ ] **I-5**: Daily digest should reflect *newly added* papers, not the raw fetch list. `collector/main.py:108` passes `all_papers` (which can re-list duplicates from prior days, especially via HF fallback). Pass `new_papers` (the diff returned by `db.append`) or filter by existing-id set in the formatter.

### Minor

- [ ] **M-1**: Live S2 verification — `S2_VENUES = ["AAAI", "NeurIPS", ...]` was never end-to-end smoke tested. Confirm S2's substring `venue` filter returns nonzero for each entry.
- [ ] **M-2**: `_short_id` collision risk for non-arXiv papers. `build_report.py:110-114` truncates `title:foo bar baz...` to 20 chars, so similar long titles produce indistinguishable refs. Use a short hash + venue prefix.
- [ ] **M-3**: Stop emitting `id` from `Paper.to_jsonl_dict` (or actually use it on read). Currently it's write-only data; if the id formula changes, old rows silently re-key and `RollingDB.append` re-adds duplicates.
- [ ] **M-4**: Document or fix `RollingDB.append` O(n²) reload-per-call. Fine at current scale, but a chunked loader would call it repeatedly; consider keeping the in-memory id set across calls.
- [ ] **M-5**: Unit-test coverage gaps:
  - `_date_window` weekend rollback (`arxiv.py:34-41`) — pure function, easy.
  - `RollingDB.append` order preservation when input list contains in-batch duplicates (current test only covers existing-on-disk dedup).
  - HF fallback loop (`huggingface.py:24-30`) — mockable session.
- [ ] **M-6**: Replace the `topic_finder ↔ topic-finder` symlink with one canonical name. Symlink invisible on Windows checkouts. Consider renaming the directory to `topic_finder` and letting SKILL.md `name: topic-finder` handle display only, or document the symlink in README.
- [ ] **M-7**: Drop `metadb/README.md` from spec §4.3 (currently referenced but not created). Since `metadb/` is gitignored, the README adds no value.
- [ ] **M-8**: Document that `Paper.categories` is currently arxiv-only. HF/OpenReview/S2 set it to `[]`. Future category-filter would silently exclude non-arXiv papers.
- [ ] **M-9**: `load_metadb.load_rolling` default path `Path("metadb/rolling.jsonl")` is relative. Either accept a `--root` flag (mirroring collector) or resolve relative to `_REPO_ROOT`.
- [ ] **M-10**: Add retention sweep to `.github/workflows/daily_collect.yml` — `metadb/daily/*.md` accumulates forever in remote repo. Sweep > 60 days before commit step.
- [ ] **M-11**: Document "first 3 of N" semantics for author truncation (`formatter.py:32`, `build_report.py:117-122`). Currently consistent but ambiguous on count of 3.
